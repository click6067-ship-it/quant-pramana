🗺️ PRAMANA — 시스템 맵 (무슨 원리로 돌아가나)V6·PAPER

한 문장: '똑똑한 AI가 종목 찍는 시스템'이 아니다. 데이터·비용 정직성으로 쉬운 알파가 없음을 먼저 입증한 뒤, 레버드 분산 베타(SPY/QQQ + managed-futures)를 변동성-반응 리스크 통제로 굴리는 솔로용 *페이퍼 검증* 시스템이다. 예측을 안 한다.

① 데이터 흐름 (매일)

📊 가격(SPY/QQQ/DBMF)→20일 변동성 측정→레버 계산(vol-target×DD-ladder·캡1.5)→북=85%레버Core+15%DBMF→next-bar 실행(비용↓)→💰 NAV→판정표·대시보드

② 레이어별 — 뭐 썼나 · 무슨 "모델" · 원리

레이어뭐 썼나모델/규칙원리(왜)
L1 데이터Sharadar(유료·backtest)·yfinance(무료·forward)PIT·survivorship-safe·가격만(V6는 펀더멘털 안 씀)과거를 *정직하게* 재현(미래 누수 차단). 깨끗한 데이터가 생존 — 더러우면 다 무의미.
L2 신호/레짐20일 실현변동성 + 현재 낙폭모델 없음 = 규칙(공식)예측 아님. *후행* 지표(변동성은 떨어진 뒤 오름). '지금 얼마나 출렁이나'만 본다.
L3 북(sleeve)85% 레버드 Core(SPY/QQQ 50/50) + 15% managed-futures(DBMF)vol-target 공식 + DD-ladder 테이블 + 고정 85/15레버=시장 노출↑(수익·리스크↑). DBMF=주식 추세하락때 *오름*→'같이 안 맞기'(분산).
L4 배분(allocator)고정 85/15 core/DBMF고정비중(적응형 allocator 아님)DeMiguel 2009: 복잡한 배분이 단순 고정비중을 OOS서 못 이긴다 → 멍청하게 고정.
L5 리스크엔진vol-target + DD-ladder + 레버캡 1.5x결정적 규칙(예측 X)변동성 터지면 노출↓·낙폭 깊으면 자동 디레버·레버 절대 1.5 초과 금지. '안 죽기' 바닥.
L6 실행next-bar(신호 t→진입 t+1)비용 모델(financing 5%·스프레드)오늘 종가 신호로 *내일* 들어감 = look-ahead(미래 훔쳐보기) 차단. 백테스트 정직성 핵심.
L7 귀속/모니터core vs DBMF 기여 분해·forward 판정표—'누가 벌었나' 분해 → 자기기만 방지. 판정표=수익만 좋으면 불합격.
L8 거버넌스paper·사람 자본게이트·LLM off-path·12mo STOP—실거래는 사람만 켤 수 있음. LLM은 자문만(주문 X). 12개월 못 맞추면 '쉬운 알파 없음' 수용.

★ 핵심: 학습된 ML/AI 모델이 0개다. 전부 *규칙·산술*(변동성 공식·낙폭 테이블·고정비중). ML/TSFM은 검증결과 알파가 없어서 의도적으로 뺐다(off-path 보조만 허용). 그래서 GPU·클라우드 불필요, CPU로 밀리초.

③ V6 북이 매일 하는 일 (이게 "원리")

  1. 어제 종가로 SPY·QQQ·DBMF 가격을 받는다.
  2. Core(SPY/QQQ 50/50)의 최근 20일 변동성을 계산한다.
  3. 레버 L = min(1.5, 28%÷변동성) — 잠잠하면 레버↑(최대 1.5x), 출렁이면 레버↓.
  4. 현재 낙폭을 보고 DD-ladder로 L을 더 줄인다(−20%↓=×0.7·−30%↓=×0.4·−40%↓=×0.2).
  5. 오늘 북 = 85%를 L배 레버한 Core + 15% DBMF (financing 비용 차감).
  6. NAV 갱신 → 판정표 채점 → 대시보드. 예측·익절·종목선택 0. 그냥 '변동성 따라 레버 조절한 분산 베타'.

④ 냉정한 평가

이게 뭔가: 정직한 레버드 분산 베타 페이퍼 북. 리스크 통제(vol-target·DD-ladder·캡1.5)로 안 죽고, 분산(DBMF)으로 상관 낙폭을 구조적으로 줄인다.

이게 아닌 것: 알파. 예측 안 함 · 시장을 *위험조정으로* 못 이김(Sharpe ≈ QQQ). 수익은 레버(리스크)에서, 낙폭통제는 분산(보험료)에서 — 공짜 없음.

강점: 가짜 알파를 한 번도 안 만들었다. 전부 정직 라벨. V6가 V5의 "같이 낙폭" 문제를 실제로 개선(6개월 진입: V5 +7.5%/−20%MDD → V6 +12.2%/−13%·12개월: QQQ도 넘김 +37.5 vs +34.1).

약점/위험: 레버 꼬리위험(forward −70%+ 가능·2008/닷컴 없는 benign 샘플) · DBMF 짧은 역사(2019~) · managed futures 2011-20 'lost decade' · forward 검증 0개월(in-sample/backtest일 뿐).

냉정한 한 줄: 돈 버는 기계도, 알파도 아니다. 안 죽고(리스크 통제) 시장을 타며(레버드 베타) 낙폭을 구조적으로 줄이는(분산) 정직한 시스템이다. in-sample 성과는 레버·분산·샘플의 결과지 *스킬*이 아니다. 진짜 가치는 forward 12개월 + 다음 크래시가 말한다. paper·NO LIVE.
