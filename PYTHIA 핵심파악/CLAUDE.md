## 🎯 북극성 (단일 목적함수 — 2026-05-29 재설정)
**pythia = 진짜 돈이 현실에서 살아남는지를 *증명*하는 leakage-safe KR 자동매매·리서치 시스템.** (정적 캡 **₩1천만**(램프 목표) + 중간복리 천장 **₩2천만** + 동적 리스크 게이트로 노출 통제. Codex 확정 2026-05-30.)

- 목표는 **오로지 수익(생존)**. 시험·발표·교수 채점은 *목적함수에서 제거됨* (발표는 부산물 — 진짜 시스템을 정직하게 보여줄 뿐).
- **성공 = "*실제 배포 가능한 자본 크기*에서, 비용(세금 0.20%·수수료·스프레드·슬리피지·미체결·현금drag) 차감 후, 패시브 벤치마크(KOSPI/KOSDAQ ETF·동일가중) 대비 초과수익."** 단순 "P&L +" 아님. 못 이기면 "못 이긴다·증거불충분"을 정직히 보고.
- **P&L 크기가 아니라 결과가 진짜임(과적합·누수 아님)이 점수.** 가치는 *자본 크기*가 아니라 검증된 edge·시스템·기술. (Codex: "자본문제가 아니라 *증명 안 된 edge* 문제.")

## 🛡 불변 원칙 (Claude×Codex 카운슬 R4 확정 — 절대 위반 금지)
1. **어떤 전략도 *실제 배포 자본 크기*에서 세금·수수료·스프레드·슬리피지·미체결·현금drag 차감 후 벤치마크를 이기기 전엔 "진짜"가 아니다.**
2. **Paper는 *운영*을 먼저 검증한다. 자본 생존은 오직 사전등록된·증액된·비용후 *실거래* 결과만 검증한다.**
3. **유니버스·피처·파라미터·재학습규칙·게이트를 결과 본 뒤 바꾸면 = 새 실험. 증거 시계 리셋** (append-only, prereg는 삭제 아니라 amend).
4. **LLM은 검사·비평·파싱·문서화만 한다. 가격예측·포지션사이징·주문승인은 절대 안 한다.**
5. **증거 불충분하면 시스템은 "증거 불충분"이라 말하고 증액을 멈춘다.**

## 프로젝트
라이브 멀티전략 퀀트 자동매매(KR, 한투 KIS, 실제 자본 캡 **₩1천만**, 중간복리 천장 ₩2천만). 상세 설계 = `docs/DESIGN.md`.

## 🔑 퀀트 규율 (= 자본 보존 갑옷. 수익 목표엔 *더* 중요 — 시장은 과적합을 진짜 돈으로 처벌)
- **No lookahead**: 모든 피처는 `available_at` 타임스탬프. 예측은 결과 윈도우 *전*에 로깅.
- **No survivorship bias**: point-in-time 유니버스 — **상폐·정지·관리/투자경고 종목 + "그때 실제 거래가능했나?" 플래그 포함** (라이브 유니버스는 "KR stocks"로 안 풀림, Codex R4).
- **No data-snooping**: 실험 사전등록(유니버스·기간·비용·train/val/test·baseline·**재학습 캘린더·파라미터 탐색공간·모델선택규칙·kill규칙·시도한 변종 개수**) 고정. 파라미터 난사 금지.
- **거래비용 항상**: 수수료·세금(매도 0.20%)·스프레드·슬리피지·체결규칙(no same-bar fill)·미체결·현금drag 반영. 안 하면 백테스트 무효.
- **baseline 필수**: buy&hold·동일가중·단일팩터·랜덤 대비 paired test + 팩터 ablation.
- **성공지표(분류정확도 아님 — 자본결과)**: 비용후 초과수익·MDD·turnover·implementation shortfall(결정가↔주문가↔체결가)·벤치 대비 Sharpe/Sortino·drawdown 지속·worst-day/week·**"비용후 벤치마크 상회 확률"**·"insufficient sample" 정직 플래그.

## 🎯 전략 제약
- **빈도 = 일~수주** (HFT·장중스캘핑·뉴스차익 ❌ = 리테일이 지는 게임, Codex 환상리스트). 장중은 사전등록 live-forward 실행 파일럿만.
- **유니버스**: **유동성 필터(거래대금·스프레드) 통과 KR — 캡 무관(대형 포함), 수익 우선** (2026-05-30: *중소형 제약 해지*, 오로지-수익 피벗 강화. 캡으로 묶지 않고 수익 나는 곳에서 고름). ⚠️ 소자본 슬리브(₩1M)는 고가주 1주 단위로 과소투입될 수 있음(2026-05-30 페이퍼 실행서 24.5% 확인; ₩5억선 99.9%) → **유니버스 안 줄이고** 자본 증액으로 표현 확장(자본은 사용자 선택).
- **유형**: 이벤트후행(실적·공시·갭·거래대금급증)·단기모멘텀+리스크필터·**조건부 참여**(좋은 셋업만 작게, 애매하면 쉼).
- **철학**: "시장 이기기"보다 **"안 불리한 전장 고르기 + 나쁜 거래 안 하기"** — 알파보다 생존.

## 🤖 AI/에이전트 = 리서처·결함감사 (예측기·트레이더 아님)

## 🛡 실제 자본 안전 (= 북극성의 핵심, 기술 컨트리뷰션)
- 🔑 **"수익목적 외 가정·한계 삭제"(용하 2026-05-30) 적용 범위**: *임의 스코프 제약*(중소형 유니버스·소액 자본 가정)은 **해지**. 그러나 *안전·방법론 갑옷*은 **유지** — 이건 임의 한계가 아니라 **수익을 *진짜·생존가능*하게 만드는 것**(지우면 가짜수익/blowup = 수익 목적과 *정반대*. Codex: 수익엔 방법론이 *더* 중요). 유지: LLM 주문권 0 · 무마진 · 일손실 서킷/킬스위치/노출캡 · no-lookahead · 비용 진실 · paper-first.
- 🚨 **자본 규율 (Codex 회의 확정 2026-05-30): 정적 캡 ₩1천만(램프 목표) + 동적 게이트 *둘 다*.** (no-cap은 Codex **UNSAFE** 판정 → KR 하한가/정지 땐 동적 stop이 *못 나가는* 순간을 막는 *대체불가 백스톱*이라 캡 복원.) **중간 복리(용하)**: 실현이익 **50% 통장 스윕(안전)·50% 보유**로 working cap을 **₩2천만 천장**까지 성장, 그 위 100% 스윕(무한복리 ❌ — 캡 무의미해지고 파멸노출↑). **전손 quit-line = 누적 −₩300만 *경제적* 손실(실현+평가, 정지종목 0마크) → 영구중단·재입금 ❌.** (시스템은 capital-agnostic — ₩5억 시드 99.9%, ₩1M선 24.5%.)
- **paper 슬리브 자본별 병렬 = *표현성 검증***(Codex R4): 같은 타겟을 ₩1M·₩10M·₩1억·₩5억 등으로 돌려 *자본별 투입률·체결성* 확인(소자본 과소투입 vs 대자본 완전표현). 유니버스/전략 안 줄이고 자본만 바꿔 봄.
- **3주 paper = "운영검증 + 초기 forward 증거"일 뿐, 지속 수익 edge 증명 아님**(불변원칙 2, Codex R4 #3).
- **실자본 증액 게이트 = 실현 체결품질**(Codex R4 #5): partial-fill율·cancel/reject율·지연·실제 스프레드·미체결 기회비용. paper 체결 ≠ 라이브 체결.
- **스타일: 캡(₩1천만→천장 ₩2천만) 안에서 성과에 공격적.** 캡 + 동적 게이트(서킷·킬·노출캡·max-DD)는 겁이 아니라 *한도 안에서 공격하게* 하는 장치. **단 결과는 진짜여야**(가짜 백테스트 수익=버그).
- 📐 **리스크 게이트 (Codex 확정 수치)**: per-name **8%**(대형 유동)/5%(코스닥150)/3%(기타)/**0%**(관리·상폐심사·정지·경고·최근하한가) · 일손실 **−2%** 신규중단 / **−3.5%** gross≤50% / **−5%** 전량청산 · **max-DD −15%** 킬(+ −10% soft, 2회=영구중단) · gross ≤95%t/≤100%h **현금전용** · **램프 ₩2M→3M→4.5M→6.75M→₩10M**(×1.5, 단계당 ≥20거래일·≥100체결·≥30종결·DD<7.5%·무위반) · **갭/유동성 사이징**(2연속 하한가 손실예산 4/2.5/1.5% → 최대 ~7.8/4.9/2.9%, ≤20일 거래대금의 2%) · 킬스위치. ✅빌트: risk.py·staging.py·capital_policy.py. 🔴데이터 후속: tier별 캡·위험플래그·ADV·MTM·정지상태(security-master+flags+ADV 소싱).
- strategy engine ↔ execution engine 분리. paper/live 동일 인터페이스. pre-trade 검증(호가단위·상하한·수량·중복 idempotency). 노출 상한(계좌·종목·섹터) · 일손실 서킷브레이커(-X% 중단, -Y% 청산) · 킬스위치 · 모든 주문 불변 감사로그.
