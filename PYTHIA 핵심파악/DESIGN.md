## 1. 북극성 (단일 목적함수)

> **pythia = 진짜 돈이 현실에서 살아남는지를 *증명*하는 leakage-safe KR 자동매매·리서치 시스템.** (정적 캡 ₩1천만(램프 목표) + 중간복리 천장 ₩2천만 + 동적 리스크 게이트. Codex 확정 2026-05-30. 시스템 자체는 capital-agnostic.)

- 목표 = **오로지 수익(생존)**. 시험·발표는 부산물(진짜 시스템을 정직하게 보여줄 뿐).
- **성공 = 실제 배포 가능한 자본 크기에서, 비용(세금 0.20%·수수료·스프레드·슬리피지·미체결·현금drag) 차감 후, 패시브 벤치마크 대비 초과수익.** 못 이기면 "못 이긴다·증거불충분" 정직 보고.
- 점수 = P&L 크기가 아니라 **결과가 진짜임(과적합·누수 아님).** "자본문제가 아니라 증명 안 된 edge 문제" — 그래서 *edge의 진실성*을 증명하는 게 본질.

### 불변 원칙 (절대)
1. 실 배포 자본 크기에서 비용 차감 후 벤치마크 못 이기면 전략은 "진짜" 아님.
2. Paper는 운영을 먼저 검증. 자본 생존은 사전등록·증액·비용후 *실거래*만 검증.
3. 유니버스·피처·파라미터·refit·게이트를 결과 본 뒤 바꾸면 새 실험 = 증거 시계 리셋(append-only).
4. LLM은 검사·비평·파싱·문서화만. 가격예측·사이징·주문승인 절대 안 함.
5. 증거 불충분 → "증거 불충분"이라 말하고 증액 중단.

## 2. 아키텍처 — 2 레이어 (분리 = 업계 정설: 알파 ↔ 실행)

### ① 리서치/검증 엔진 (LLM은 *여기서만*, 비평·감사 역할)
```
전략 아이디어
  → 카운슬(결함 감사: 누수·비용무시·과적합·저유동)
  → 결정론 게이트(사전등록 임계값, LLM 아님)
  → walk-forward 백테스트(PIT·비용반영)
  → 목표비중(target weights)
```
- **카운슬 = 자본방어 필터**(상세 `COUNCIL.md`). 결정론 체크리스트를 못 이기는 역할은 삭제.
- **모든 LLM 호출은 하류 결정론 게이트가 소비하는 persisted JSON artifact를 내야 한다. 안 내면 컷.**

### ② 실행 엔진 (KIS, 결정론, LLM 0)
```
목표비중 → 잔고 조회 → 차이만 주문 → pre-trade 검증 → 체결
         → 노출캡·일손실 서킷·킬스위치 → 불변 감사로그
```
- 주문 트리거·차단 100% 결정론. paper/live 동일 인터페이스.
- OMS 축소판(주문 상태기계 + 멱등성 + 체결기록)은 실행엔진 구현 시 필수. EMS/실행알고는 소자본이라 최소화.

### 빈도 (미니PC 24시간 운영)
- **② 실행** = 장중 상시(가벼움, LLM 없음). **① 리밸런스 계산** = 일/주 1회(순수 규칙). **① 카운슬(LLM)** = 새 전략 검증 시에만(드묾). LLM은 claude/codex CLI 구독인증(API키 없음, bwrap 격리).

## 3. 데이터 (자본 보존 갑옷)
- KRX(pykrx)·DART·FinanceDataReader → 로컬 Parquet 캐시. 모든 데이터 `available_at` 강제.
- **PIT 유니버스 = 상폐·정지·관리/투자경고 종목 + "그때 실제 거래가능?" 플래그 포함** (라이브 생존편향 차단).
- 유동성 필터 강함(거래대금·스프레드) → 1주 단위·현금drag 지옥 회피.

## 4. 팩터·전략
- 가격 팩터: 모멘텀·저변동·단기역전 등. winsorize + z-score + 동일가중 결합. IC/turnover 오프라인 진단.
- 전략: top-N 횡단면(일봉) = 결정 객체. 이벤트후행·단기모멘텀+리스크필터·조건부 참여.
- 빈도 일~수주. 인버스 ETF/현금으로 시장중립 근사(개인 공매도 제약).

## 5. 백테스터 (가혹한 현실)
- 체결 = **다음 날 시가(no same-bar fill)**. 수수료·매도세 0.20%·스프레드·슬리피지 차감.
- walk-forward: 규칙 freeze, 파라미터만 사전등록 캘린더로 재적합. **재학습 캘린더·피처·파라미터공간·모델선택·kill규칙·시도 변종수 사전등록**(refit-snooping 차단).
- baseline: buy&hold·동일가중·단일팩터·랜덤 + 느린실행(next-close) 벤치마크.

## 6. 자본 사다리 (paper → 소액 → 증액)
- **paper 슬리브 자본별 병렬**: ₩1M·₩2M·₩10M(실 한도 매칭) + ₩1억은 *용량 스트레스용만*. (단일 ₩1억 paper로 소액 검증 금지.)
- 실: paper 운영검증 → **램프 ₩2M→3M→4.5M→6.75M→₩10M**(×1.5, 단계당 ≥20거래일·100체결·30종결·DD<7.5%·무위반) → 정적 캡 ₩1천만 도달. **중간복리**: 실현이익 50% 스윕·50% 보유로 working cap ₩2천만 천장까지, 그 위 100% 스윕. 전손 quit −₩300만 경제적. 노출 통제 = 동적 게이트(§9) + max-DD −15% + ADV 사이징.
- **증액 게이트 = 실현 체결품질**: partial-fill·cancel/reject·지연·실제 스프레드·미체결 기회비용. + 비용후 벤치 대비 결과.
- 3주 paper = "운영검증 + 초기 forward 증거"일 뿐(지속 edge 증명 아님).

## 7. 성공 지표 (자본 결과 — 분류정확도 아님)
비용후 초과수익 · MDD · turnover · implementation shortfall · 벤치 대비 Sharpe/Sortino · drawdown 지속 · worst-day/week · **비용후 벤치 상회 확률** · "insufficient sample" 플래그. 카운슬 가치 = 결함적발·예방손실·latency/비용.

## 8. 대시보드 (증거 뷰어)

- **No-lookahead provenance**: D 결정 → D+1 체결 갭을 로그로 증명.
