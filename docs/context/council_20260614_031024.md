**≤6 Objections**
1. **[발견] A1 인셉션 불일치**  
   왜 실패: PROJECT_STATE는 라이브 인셉션 `2026-06-12`인데, 실제 `state.json`/공개 `docs/index.html`은 `2026-05-12`를 표시.  
   임팩트: “forward 관찰”과 “backfilled paper” 경계가 다시 흐려진다.  
   수정안: 루트와 iframe에 `Live inception=2026-06-12`, `Backfill window=2026-05-12~2026-06-11`를 분리 표기.

2. **[발견] LIVE 배지가 아직 과하다**  
   왜 실패: 상단은 `PAPER`로 고쳤지만 nav/섹션/iframe은 `A1 Attack (LIVE)`, `LIVE` 빨간 배지가 시각적으로 더 강하다. SEC/Investor.gov도 투자판단은 공식·검증된 정보 확인을 요구하는 톤이고, 8-K 같은 material event도 정해진 current report 체계로 다룬다. SEC Form 8-K는 current report이며 중요 사건은 보통 4영업일 내 보고된다.   
   임팩트: 교수/외부 독자는 “실계좌 라이브”로 오독 가능.  
   수정안: `LIVE` → `PAPER FORWARD` 또는 `SIM LIVE FEED`; 빨간 배지는 `NO CAPITAL`.

3. **[발견] Evidence Ledger는 좋지만 proof가 없다**  
   왜 실패: `DSR/PBO TODO`, `2-feed UNKNOWN`, `cron 미검증`은 보이나 클릭 가능한 receipt/health/log 링크가 없다.  
   임팩트: 정직한 고백이 “검증 가능한 증거”로 닫히지 않는다.  
   수정안: Ledger 각 항목 옆에 `health.json`, `nav_log.csv`, council md, report md 링크를 붙여라.

4. **[발견] backtest→live 할인 근거가 화면에 부족**  
   왜 실패: 차트는 여전히 2016~2026 구조성과가 첫 인상이다. 최신 backtest 연구도 marketed/pro-forma 성과는 live로 가면 약해지고 peer/external benchmark 기준에서 더 약해진다고 보고한다. ([arxiv.org](https://arxiv.org/abs/2604.18821))  
   임팩트: “좋은 대시보드”가 다시 “좋은 전략”처럼 보인다.  
   수정안: 첫 화면 차트 제목에 `IN-SAMPLE STRUCTURE ONLY`를 박고, live KPI를 차트 위로 올리지 말 것.

5. **[발견] A1 iframe KPI가 실계좌 UI처럼 생겼다**  
   왜 실패: `총 NAV`, `총수익`, `vs QQQ` KPI가 가장 큼. 아래 경고문은 읽어야 보인다.  
   임팩트: 정직성 점수 상한을 친다.  
   수정안: KPI 카드 라벨을 `Paper NAV`, `Paper Return`, `Capital Deployed: ₩0`로 바꾸고 `NO BROKER / NO ORDER` 카드를 추가.

6. **[발견] 알파 난이도 외부 prior는 충분히 안 보인다**  
   왜 실패: “scope-conditional negative”는 좋아졌지만, 왜 그 결론이 합리적인지 외부 base-rate가 약하다. SPIVA 2025는 active large-cap U.S. equity funds의 **79%**가 S&P 500을 underperform했다고 보고한다. ([spglobal.com](https://www.spglobal.com/spdji/en/spiva/article/spiva-us/))  
   임팩트: 교수 입장에서는 “내 프로젝트만 실패”인지 “시장적으로 어려운 문제를 잘 기각”했는지 덜 선명하다.  
   수정안: Evidence Ledger 또는 overview 하단에 SPIVA/active base-rate 한 줄 추가.
